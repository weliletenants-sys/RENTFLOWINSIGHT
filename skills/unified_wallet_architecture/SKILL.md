# Unified Wallet Architecture

RentFlowInsight uses a fully unified wallet architecture to manage user balances across all their different operational roles (Personas).

## Core Concepts

### 1. A Single "True" Wallet per User
When a user registers, the system creates exactly **one** record in the `wallets` table tied to their central `profile.id`. There are no separate "Funder Wallets" or "Agent Wallets" at the database level. 

### 2. Logical Separation via `walletBuckets`
To track where the money actually is across the different personas without creating separate wallets, that single wallet has 5 distinct buckets attached to it:
- `available` (Liquid cash ready to withdraw or invest)
- `invested` (Capital currently locked in the rent pool)
- `commission` (Earnings generated by the Agent persona)
- `reserved` 
- `savings`

### 3. Unified Personas (`userPersonas`)
A user can hold multiple roles simultaneously (e.g., they can be both a `FUNDER` and an `AGENT`). Because they share the same central `wallet.id`, an Agent who earns commissions into their `commission` bucket can easily transfer that to their `available` bucket and seamlessly invest it as a Funder into the rent pool—all without needing to withdraw to an external bank and re-deposit into the system.

## Summary
The system is fully unified across personas, using the bucket architecture to keep the accounting strictly organized under one roof while providing maximum flexibility for users handling multiple roles on the platform. It allows users to quickly maneuver their capital across the ecosystem without dealing with completely disconnected ledgers.
